# Discovery Synthesis Reference

## How to Write a Discovery Summary

After collecting answers from the operator, synthesize into this format:

```
DISCOVERY SUMMARY
─────────────────────────────────────────────────────────
Product Idea:    [One sentence. What the product does.]
Core Problem:    [One sentence. The specific pain being solved.]
Target User:     [One sentence. Who they are and their context.]
Key Insight:     [One sentence. The non-obvious angle or opportunity.]
Platform:        [mobile / web / both / TBD]
Constraints:     [Any known constraints: timeline, tech, budget]
─────────────────────────────────────────────────────────
```

---

## Example 1 — Coffee Shop Loyalty App

```
DISCOVERY SUMMARY
─────────────────────────────────────────────────────────
Product Idea:    A mobile app that lets independent coffee shop owners run
                 digital loyalty programs without needing a developer.
Core Problem:    Small coffee shops lose repeat customers because they rely
                 on paper punch cards that customers forget or lose.
Target User:     Independent coffee shop owners (1–3 locations) in the US
                 who have no dedicated IT support.
Key Insight:     Square and Toast are overkill; there's no lightweight,
                 mobile-first loyalty product built specifically for indie
                 coffee shops.
Platform:        Mobile (iOS first), with a web admin panel
Constraints:     MVP in 3 months, no backend team, solo founder
─────────────────────────────────────────────────────────
```

## Example 2 — Freelancer Invoice Tracker

```
DISCOVERY SUMMARY
─────────────────────────────────────────────────────────
Product Idea:    A simple web app that helps freelancers track unpaid
                 invoices and send automated follow-up reminders.
Core Problem:    Freelancers forget to chase late payments and lose
                 income because chasing clients feels awkward.
Target User:     Freelance designers, writers, and developers earning
                 $50k–$150k/year who manage invoicing manually.
Key Insight:     Existing tools like FreshBooks are full accounting suites.
                 There's no product focused solely on the "get paid" workflow.
Platform:        Web app
Constraints:     Stripe integration required, launch in 6 weeks
─────────────────────────────────────────────────────────
```

## Quality Check Before Handing Off

Before passing the Discovery Summary to the next skill, confirm:

- [ ] Product idea is software or app (not physical, not service)
- [ ] Problem is specific — not "people want to be more productive"
- [ ] Target user is specific — not "everyone" or "small businesses"
- [ ] At least one constraint or platform preference is noted
- [ ] Key insight identifies the differentiation angle or why now

If any check fails, ask one follow-up question to fill the gap.
