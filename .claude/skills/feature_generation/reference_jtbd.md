# Jobs To Be Done (JTBD) Framework

## What is a JTBD?

A Job To Be Done describes what a user is trying to accomplish in a given situation — independent of the solution. It captures motivation and outcome, not just action.

**Format:**
> "When [situation], I want to [motivation], so I can [outcome]."

---

## How to Extract JTBDs from a Discovery Summary

Ask these questions about the target user:

1. What is the recurring situation that triggers them to need your product?
2. What are they really trying to accomplish (not just the surface task)?
3. What does success look like for them after using the product?
4. What anxiety or fear are they trying to avoid?
5. What social or professional outcome do they want?

Typically 3–5 strong JTBDs emerge per product. Do not pad to more than 6.

---

## JTBD Quality Check

Strong JTBDs:
- Describe a real, recurring situation
- Are solution-agnostic (not tied to your specific product)
- Have a clear desired outcome
- Would generate different features than a generic problem statement

Weak JTBDs:
- "I want to use the app" — too vague
- "I want to see my dashboard" — too feature-specific
- "I want a loyalty program" — solution-first, not problem-first

---

## Example JTBDs — Coffee Shop Loyalty App

**JTBD 1**
When a customer is at the counter, I want to track their visit quickly, so I can reward loyalty without slowing down the line.

**JTBD 2**
When I'm planning a slow week, I want to send a targeted promotion to my regulars, so I can drive traffic without spending money on ads.

**JTBD 3**
When a customer hasn't visited in 3 weeks, I want to automatically re-engage them, so I don't lose them to the coffee shop next door.

**JTBD 4**
When I'm too busy to manage tech, I want to set up rewards once and have them run automatically, so I can focus on making coffee.

**JTBD 5**
When I'm reviewing the week, I want to see which customers are my most loyal, so I can give them special treatment and strengthen the relationship.

---

## From JTBDs to Features

For each JTBD, ask:
- What feature(s) would make this job trivially easy to accomplish?
- What does the user need to see or do to accomplish this job?
- What would remove the most friction in accomplishing this job?

Example mapping:

| JTBD | Features Generated |
|---|---|
| Track visits quickly at counter | QR code check-in, one-tap stamp, POS-mode screen |
| Send targeted promotions | Segment by visit frequency, push notification campaigns |
| Re-engage lapsed customers | Automated "we miss you" notifications, win-back offers |
| Run automatically without management | Reward rules engine, auto-trigger by visit count |
| See loyalty analytics | Customer visit history, loyalty leaderboard, export CSV |
